Arquitectura del Proyecto

Antes de ejecutar cualquier comando es fundamental entender qué estamos construyendo y cómo se comunican los componentes.

00
Topología Master + Workers (Clúster distribuido)
ARQUITECTURA

El proyecto usa un clúster de 4 nodos: 1 Master y 3 Workers. El nodo Master coordina todo; los Workers procesan y almacenan datos.

03
Configurar SSH sin contraseña (Master → Workers)
CONFIGURACIÓN

Hadoop y Spark necesitan comunicarse entre nodos sin pedir contraseña. Generamos un par de llaves SSH y las copiamos a todos los nodos.

BASH — En TODOS los nodos: instalar SSH 
# Instalar servidor SSH (para recibir conexiones)
sudo apt install -y openssh-server openssh-client

# Iniciar el servicio SSH
sudo systemctl enable ssh
sudo systemctl start ssh
BASH — Solo en el MASTER: generar y distribuir llaves 
# Cambiar al usuario hadoop
su - hadoop

# Generar par de llaves RSA
# ssh-keygen → genera el par clave pública/privada
# -t rsa → tipo de cifrado RSA
# -b 4096 → longitud de 4096 bits (más seguro)
# -C → comentario identificador
# Cuando pregunte passphrase → presionar ENTER (sin contraseña)
ssh-keygen -t rsa -b 4096 -C "hadoop@master"

# Copiar la llave pública a todos los nodos (incluyendo el master mismo)
# ssh-copy-id → copia la clave pública al archivo authorized_keys del destino
ssh-copy-id hadoop@master
ssh-copy-id hadoop@worker1
ssh-copy-id hadoop@worker2
ssh-copy-id hadoop@worker3

# Verificar que funciona (debe entrar SIN pedir contraseña)
ssh hadoop@worker1
exit
ssh hadoop@worker2
exit
✓
Si al ejecutar ssh hadoop@worker1 entras directamente sin contraseña, el SSH está configurado correctamente.

07
Configurar los archivos XML de Hadoop
CONFIGURACIÓN

Hadoop se configura mediante 4 archivos XML. Estos van en /opt/hadoop/etc/hadoop/ y deben ser idénticos en TODOS los nodos.

core-site.xml — Define el sistema de archivos principal (HDFS) y su dirección.

XML — /opt/hadoop/etc/hadoop/core-site.xml 
<configuration>
  <property>
    <!-- fs.defaultFS → URI del sistema de archivos por defecto -->
    <!-- hdfs://master:9000 → usa HDFS en el nodo "master" puerto 9000 -->
    <name>fs.defaultFS</name>
    <value>hdfs://master:9000</value>
  </property>
  <property>
    <!-- hadoop.tmp.dir → directorio temporal de Hadoop -->
    <name>hadoop.tmp.dir</name>
    <value>/home/hadoop/hadooptmpdata</value>
  </property>
</configuration>

hdfs-site.xml — Configura HDFS: factor de replicación de bloques y rutas de almacenamiento.

XML — /opt/hadoop/etc/hadoop/hdfs-site.xml 
<configuration>
  <property>
    <!-- dfs.replication → cuántas copias guarda HDFS de cada bloque -->
    <!-- 3 = ideal con 3 workers (una copia por nodo) -->
    <name>dfs.replication</name>
    <value>3</value>
  </property>
  <property>
    <!-- namenode.name.dir → dónde el NameNode guarda los metadatos -->
    <name>dfs.namenode.name.dir</name>
    <value>file:///home/hadoop/hdfs/namenode</value>
  </property>
  <property>
    <!-- datanode.data.dir → dónde los DataNodes guardan los bloques -->
    <name>dfs.datanode.data.dir</name>
    <value>file:///home/hadoop/hdfs/datanode</value>
  </property>
</configuration>

mapred-site.xml — Configura MapReduce para usar YARN como gestor de recursos.

XML — /opt/hadoop/etc/hadoop/mapred-site.xml 
<configuration>
  <property>
    <!-- mapreduce.framework.name → framework de procesamiento -->
    <!-- "yarn" → usa YARN para gestionar la asignación de recursos -->
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
  </property>
</configuration>

yarn-site.xml — Configura YARN: gestor de recursos del clúster.

XML — /opt/hadoop/etc/hadoop/yarn-site.xml 
<configuration>
  <property>
    <!-- yarn.resourcemanager.hostname → nodo donde corre el ResourceManager -->
    <name>yarn.resourcemanager.hostname</name>
    <value>master</value>
  </property>
  <property>
    <!-- nodemanager.aux-services → servicio auxiliar para shuffle de MapReduce -->
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
  </property>
  <property>
    <!-- nodemanager.aux-services.mapreduce.shuffle.class -->
    <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name>
    <value>org.apache.hadoop.mapred.ShuffleHandler</value>
  </property>
</configuration>
BASH — Configurar archivo workers (solo en MASTER) 
# workers → lista de nodos que actuarán como DataNode y NodeManager
# Hadoop inicia/detiene servicios en estos nodos remotamente via SSH
nano /opt/hadoop/etc/hadoop/workers

# Contenido del archivo (eliminar "localhost" y poner:
worker1
worker2
worker3
BASH — Copiar configs a todos los workers (desde MASTER) 
# scp → copia segura de archivos vía SSH
# -r → recursivo (copia toda la carpeta)
# Copiamos toda la configuración de hadoop a cada worker
scp -r /opt/hadoop hadoop@worker1:/opt/
scp -r /opt/hadoop hadoop@worker2:/opt/

08
Formatear NameNode e Iniciar Hadoop
EJECUCIÓN

El formateo del NameNode inicializa el sistema de archivos HDFS. Solo se hace UNA VEZ. Luego iniciamos los servicios HDFS y YARN.

⚠
¡IMPORTANTE! El formateo del NameNode solo se ejecuta la primera vez. Si lo vuelves a ejecutar perderás todos los datos en HDFS.
BASH — Solo en MASTER 
# Crear los directorios de datos HDFS (en TODOS los nodos)
# mkdir -p → crea directorios incluyendo los padres si no existen
mkdir -p /home/hadoop/hdfs/namenode  # Solo en MASTER
mkdir -p /home/hadoop/hdfs/datanode   # En workers también
mkdir -p /home/hadoop/hadooptmpdata   # En todos los nodos

# FORMATEAR el NameNode (solo primera vez, solo en MASTER)
# namenode -format → inicializa las estructuras del sistema de archivos HDFS
# Responder "Y" cuando pregunte si deseas formatear
hdfs namenode -format

# Iniciar el sistema de archivos distribuido HDFS
# start-dfs.sh → arranca: NameNode (master) + DataNodes (workers) + SecondaryNameNode
start-dfs.sh

# Iniciar YARN (gestor de recursos)
# start-yarn.sh → arranca: ResourceManager (master) + NodeManagers (workers)
start-yarn.sh

# Verificar los procesos corriendo con jps (Java Process Status)
# jps → muestra todos los procesos Java en ejecución
# En el MASTER deberías ver:
# NameNode, SecondaryNameNode, ResourceManager
jps

# Verificar en un worker (también con jps):
# En cada WORKER deberías ver:
# DataNode, NodeManager
ssh worker1 "jps"

Python — Pipeline ETL completo (guardar como pipeline.py) 
#!/usr/bin/env python3
# pipeline.py — ETL para generar kpi_sales.csv desde HDFS

from pyspark.sql import SparkSession
from pyspark.sql.functions import col, sum, avg, count, when, round

# ── 1. Crear la sesión Spark ────────────────────────────────────
# SparkSession.builder → constructor de la sesión
# .master() → conectar al Spark Master del clúster
# .appName() → nombre identificador del job (visible en Spark UI)
spark = SparkSession.builder \
    .master("spark://master:7077") \
    .appName("RetailMax_ETL") \
    .getOrCreate()

# ── 2. Leer el dataset desde HDFS ──────────────────────────────
df = spark.read.csv(
    "hdfs://master:9000/retail_project/retail_sales_dataset.csv",
    header=True,
    inferSchema=True
)

# ── 3. Limpiar y preparar datos ────────────────────────────────
# Seleccionar y renombrar columnas relevantes
# cast("double") → convertir a número decimal si viene como string
df_clean = df.select(
    col("transaction_id"),
    col("date"),
    col("product_name"),
    col("category"),
    col("region"),
    col("sales_channel"),
    col("customer_segment"),
    col("customer_age_group"),
    col("payment_method"),
    col("sales_amount").cast("double"),
    col("discount_pct").cast("double"),
    col("units_sold").cast("integer")
).dropna()

# ── 4. Distribuir en particiones ───────────────────────────────
# repartition() → redistribuye para procesamiento paralelo
df_clean = df_clean.repartition(7)

# ── 5. Convertir a pandas para guardar como CSV local ──────────
# toPandas() → convierte el Spark DataFrame a pandas
# NOTA: solo es viable porque el resultado final es pequeño
df_pandas = df_clean.toPandas()

# ── 6. Guardar en el sistema local del master ──────────────────
import os
os.makedirs("/home/hadoop/data", exist_ok=True)
df_pandas.to_csv("/home/hadoop/data/kpi_sales.csv", index=False)

print("✓ Pipeline completado: kpi_sales.csv generado")
print(f"  Registros: {len(df_pandas):,}")

spark.stop()

#!/usr/bin/env python3
# app.py — Dashboard Ejecutivo RetailMax
# Requiere: streamlit, pandas, plotly

import streamlit as st
import pandas as pd
import plotly.express as px
import plotly.graph_objects as go

# ── CONFIG DE PÁGINA ────────────────────────────────────────────
# set_page_config → configura la página antes de cualquier elemento
# page_title → título en la pestaña del navegador
# layout="wide" → usa todo el ancho disponible (mejor para dashboards)
# page_icon → emoji o ruta a ícono
st.set_page_config(
    page_title="RetailMax Dashboard",
    layout="wide",
    page_icon="📊"
)

# ── ESTILOS GLOBALES PARA PLOTLY ────────────────────────────────
# Configuración base de layout para todos los gráficos
# font color="black" → texto negro visible sobre fondo blanco
PLOTLY_LAYOUT = {
    "font": dict(color="black", size=12),
    "xaxis": dict(
        tickfont=dict(color="black", size=11),
        titlefont=dict(color="black", size=12)
    ),
    "yaxis": dict(
        tickfont=dict(color="black", size=11),
        titlefont=dict(color="black", size=12)
    ),
    "plot_bgcolor": "white",
    "paper_bgcolor": "white",
    "legend": dict(font=dict(color="black"))
}

# ── TÍTULO ──────────────────────────────────────────────────────
st.title("📊 RetailMax — Executive Analytics Dashboard")
st.markdown("**Dataset:** 120,000 transacciones | **Procesado con:** Apache Spark 3.5.1")

# ── CARGAR DATOS ────────────────────────────────────────────────
# @st.cache_data → cachea los datos en memoria para no releer el CSV en cada interacción
# La función solo se ejecuta la primera vez o cuando cambia el archivo
@st.cache_data
def load_data():
    df = pd.read_csv("/home/hadoop/data/kpi_sales.csv")
    # Convertir columnas numéricas (por si vienen como string)
    df["sales_amount"] = pd.to_numeric(df["sales_amount"], errors="coerce")
    df["discount_pct"] = pd.to_numeric(df["discount_pct"], errors="coerce")
    df["units_sold"]   = pd.to_numeric(df["units_sold"],   errors="coerce")
    return df

df = load_data()

st.divider()

# ════════════════════════════════════════════════════════════════
# SECCIÓN 1: KPIs PRINCIPALES (FILA DE MÉTRICAS)
# st.metric() → tarjeta KPI con valor y delta (variación)
# ════════════════════════════════════════════════════════════════
st.subheader("📈 KPIs Principales")

k1, k2, k3, k4, k5 = st.columns(5)

# KPI 1: Ventas Totales
k1.metric("💰 Ventas Totales", f"S/ {df['sales_amount'].sum():,.0f}")

# KPI 2: Ticket Promedio (valor medio por transacción)
k2.metric("🧾 Ticket Promedio", f"S/ {df['sales_amount'].mean():,.2f}")

# KPI 3: Total Transacciones
k3.metric("🔢 Transacciones", f"{len(df):,}")

# KPI 4: Descuento Promedio
k4.metric("🏷️ Descuento Prom.", f"{df['discount_pct'].mean():.1f}%")

# KPI 5: Total Unidades Vendidas
k5.metric("📦 Unidades Vendidas", f"{int(df['units_sold'].sum()):,}")

st.divider()

# ════════════════════════════════════════════════════════════════
# SECCIÓN 2: GRÁFICOS DE VENTAS (FILA 1)
# ════════════════════════════════════════════════════════════════
c1, c2 = st.columns(2)

# KPI 6: Ventas por Categoría — Gráfico de Barras
with c1:
    st.subheader("📦 KPI 6: Ventas por Categoría")
    cat = df.groupby("category")["sales_amount"].sum().reset_index()
    fig = px.bar(cat, x="category", y="sales_amount",
                color="category", title="Ventas Totales por Categoría")
    fig.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig, use_container_width=True)

# KPI 7: Ventas por Región — Gráfico de Líneas
with c2:
    st.subheader("📍 KPI 7: Ventas por Región")
    reg = df.groupby("region")["sales_amount"].sum().reset_index()
    fig2 = px.line(reg, x="region", y="sales_amount",
                  markers=True, title="Ventas por Región")
    fig2.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig2, use_container_width=True)

# ════════════════════════════════════════════════════════════════
# SECCIÓN 3: GRÁFICOS DE CANAL Y SEGMENTO (FILA 2)
# ════════════════════════════════════════════════════════════════
c3, c4 = st.columns(2)

# KPI 8: Ventas por Canal — Pie Chart
with c3:
    st.subheader("🛒 KPI 8: Ventas por Canal")
    ch = df.groupby("sales_channel")["sales_amount"].sum().reset_index()
    fig3 = px.pie(ch, names="sales_channel", values="sales_amount",
                  title="Distribución de Ventas por Canal")
    fig3.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig3, use_container_width=True)

# KPI 9: Segmento de Cliente — Barras horizontales
with c4:
    st.subheader("👥 KPI 9: Segmento de Cliente")
    seg = df.groupby("customer_segment")["sales_amount"].sum().reset_index()
    fig4 = px.bar(seg, x="sales_amount", y="customer_segment",
                orientation="h", title="Ventas por Segmento de Cliente",
                color="customer_segment")
    fig4.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig4, use_container_width=True)

st.divider()

# ════════════════════════════════════════════════════════════════
# SECCIÓN 4: TOP PRODUCTOS
# ════════════════════════════════════════════════════════════════
st.subheader("🏆 KPI 10: Top 10 Productos por Ventas")
top10 = (df.groupby("product_name")["sales_amount"]
            .sum()
            .sort_values(ascending=False)
            .head(10)
            .reset_index())

fig5 = px.bar(top10, x="sales_amount", y="product_name",
             orientation="h", title="Top 10 Productos",
             color="sales_amount", color_continuous_scale="Greens")
fig5.update_layout(**PLOTLY_LAYOUT)
st.plotly_chart(fig5, use_container_width=True)

st.divider()

# ════════════════════════════════════════════════════════════════
# SECCIÓN 5: ANÁLISIS ADICIONAL
# ════════════════════════════════════════════════════════════════
c5, c6 = st.columns(2)

with c5:
    st.subheader("💳 Métodos de Pago")
    pay = df.groupby("payment_method")["sales_amount"].sum().reset_index()
    fig6 = px.pie(pay, names="payment_method", values="sales_amount",
                  hole=0.4, title="Distribución por Método de Pago")
    fig6.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig6, use_container_width=True)

with c6:
    st.subheader("🎂 Ventas por Grupo de Edad")
    age = df.groupby("customer_age_group")["sales_amount"].sum().reset_index()
    fig7 = px.area(age, x="customer_age_group", y="sales_amount",
                   title="Ventas por Grupo de Edad")
    fig7.update_layout(**PLOTLY_LAYOUT)
    st.plotly_chart(fig7, use_container_width=True)

# ────────────────────────────────────────────────────────────────
# TABLA PIVOT: Descuento por Categoría y Canal
# ────────────────────────────────────────────────────────────────
st.subheader("📊 Descuento Promedio por Categoría y Canal")
pivot = df.pivot_table(
    index="category",
    columns="sales_channel",
    values="discount_pct",
    aggfunc="mean"
)
# st.dataframe() → muestra una tabla interactiva con scroll y ordenamiento
st.dataframe(pivot.style.format("{:.2f}%"), use_container_width=True)

Errores Comunes y Cómo Resolverlos

Todos los errores que surgieron durante el desarrollo del proyecto y su solución exacta.

⚠ ERROR 01 — ModuleNotFoundError: pyspark
ModuleNotFoundError: No module named 'pyspark'
✓ Causa
El archivo app.py tenía from pyspark.sql import SparkSession dentro del entorno virtual de Streamlit, que no tiene PySpark instalado.
✓ Solución
Streamlit NO debe usar PySpark directamente. El pipeline de Spark genera kpi_sales.csv y Streamlit lo lee con pandas. Eliminar cualquier import pyspark del app.py.
Python — CORRECTO en app.py 
# ✓ CORRECTO: solo pandas para leer el CSV
import pandas as pd
df = pd.read_csv("/home/hadoop/data/kpi_sales.csv")

# ✗ INCORRECTO: no poner esto en app.py
# from pyspark.sql import SparkSession
⚠ ERROR 02 — Workers DEAD en Spark UI
DEAD worker1, worker2, worker3 — Status: DEAD
✓ Causa
Los workers de Spark se cayeron por inactividad, falta de memoria o timeout de heartbeat.
✓ Solución
BASH — Reiniciar workers Spark 
# Detener todos los workers
$SPARK_HOME/sbin/stop-workers.sh

# Reiniciar workers
$SPARK_HOME/sbin/start-workers.sh

# Verificar en http://master:8080 que aparezcan como ALIVE
# Si persiste, reiniciar todo el clúster Spark:
$SPARK_HOME/sbin/stop-all.sh
$SPARK_HOME/sbin/start-all.sh
⚠ ERROR 03 — JAVA_HOME no definido
Error: JAVA_HOME is not set and could not be found.
✓ Causa
La variable JAVA_HOME no está definida en la sesión actual.
BASH — Solución 
# Verificar que está en ~/.bashrc
grep JAVA_HOME ~/.bashrc

# Si no aparece, agregarlo:
echo 'export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64' >> ~/.bashrc

# Recargar configuración
source ~/.bashrc

# Verificar
echo $JAVA_HOME

KeyError: 'customer_segment'

El CSV generado por PySpark no tiene esa columna, o el nombre es diferente (espacios, mayúsculas).
Python — Diagnóstico

Ajustar el nombre en app.py a como aparezca exactamente (incluyendo mayúsculas y underscores).
⚠ ERROR 05 — DataNodes no conectan al NameNode
java.io.IOException: Incompatible clusterIDs
✓ Causa
Se formateó el NameNode más de una vez, generando un nuevo clusterID que no coincide con el que tienen los DataNodes.
BASH — Solución 
# Detener Hadoop
stop-dfs.sh

# Limpiar datos de DataNodes EN CADA WORKER
ssh worker1 "rm -rf /home/hadoop/hdfs/datanode/*"
ssh worker2 "rm -rf /home/hadoop/hdfs/datanode/*"
ssh worker3 "rm -rf /home/hadoop/hdfs/datanode/*"

# Limpiar datos del NameNode
rm -rf /home/hadoop/hdfs/namenode/*

# Re-formatear el NameNode
hdfs namenode -format

# Reiniciar HDFS
start-dfs.sh
⚠ ERROR 06 — Gráficos con texto invisible (gris sobre blanco)
Ejes y etiquetas de Plotly no se ven en Streamlit (color gris muy claro)
✓ Causa
Plotly aplica un tema automático con bajo contraste que mezcla el texto con el fondo blanco de Streamlit.
✓ Solución
Aplicar el objeto PLOTLY_LAYOUT global con font color="black" en todos los gráficos mediante fig.update_layout(**PLOTLY_LAYOUT).
